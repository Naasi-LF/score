## 1. 启动系统

### 首次启动

```bash
# 1. 启动所有服务
docker compose up --build -d

# 2. 将数据集放入指定目录
#    文件路径: backend/ml/datasets/student-mat.csv
#    数据集来源: UCI Student Performance Dataset

# 3. 训练机器学习模型（必须在有数据集后执行）
docker compose exec backend python ml/train_all.py
```

训练完成后，终端会输出每个模型的评估指标，模型文件保存在 `backend/ml/saved_models/`。

### 日常启动

```bash
docker compose up -d
```

### 访问地址

| 服务 | 地址 |
|------|------|
| 前端页面 | http://localhost:3000 |
| 后端 API 文档 | http://localhost:8000/docs |

## 3. 数据概览

登录后默认进入「数据概览」页面，展示：

- **学生总数** — 数据库中的学生记录数量
- **期中平均分 G1** — 所有学生第一次期中成绩均值
- **期末平均分 G2** — 所有学生第二次期中成绩均值
- **高风险人数** — 被判定为高风险的学生数量
- **成绩分布饼图** — 各成绩区间（优/良/中/差/不及格）占比
- **成绩区间柱状图** — 各区间人数对比

> 数据概览依赖数据库中已有学生数据。如果刚启动系统，需先在「学生管理」中导入数据。

## 4. 学生管理

点击左侧导航「学生管理」进入。

### 4.1 查看学生列表

页面默认展示所有学生，每页 20 条，支持翻页。表格字段说明：

| 字段 | 说明 |
|------|------|
| ID | 系统自动分配的唯一编号 |
| 姓名 | 学生姓名 |
| 年龄 | 学生年龄 |
| 性别 | M = 男，F = 女 |
| G1 | 第一次期中成绩（0–20） |
| G2 | 第二次期中成绩（0–20） |
| 学习时长 | 每周学习时长等级（1–4） |
| 不及格次数 | 历史不及格次数（0–4） |
| 缺勤 | 缺勤天数 |

### 4.2 新增学生

1. 点击「＋ 新增学生」按钮
2. 在弹出的表单中填写各项信息：
   - 姓名、年龄、性别
   - 住址（城市 U / 农村 R）
   - 家庭规模（≤3人 LE3 / >3人 GT3）
   - 课外活动（有 / 无）
   - 母亲学历、父亲学历（0–4，0=无学历，4=高等教育）
   - 学习时长（1=<2小时，2=2–5小时，3=5–10小时，4=>10小时）
   - 不及格次数、缺勤次数
   - G1 成绩、G2 成绩
3. 点击「确认添加」

### 4.3 批量导入

1. 点击「↑ 批量导入 CSV」按钮
2. 选择 `.csv` 或 `.xlsx` 文件
3. 系统自动解析并导入，完成后提示导入条数

CSV 文件格式要求（第一行为表头）：

```
name,age,sex,address,famsize,medu,fedu,studytime,failures,absences,activities,g1,g2
张三,17,M,U,GT3,2,2,2,0,3,yes,12,11
```

### 4.4 删除学生

点击对应行的「删除」按钮，确认后删除该学生记录。

## 5. 成绩预测

点击左侧导航「成绩预测」进入，对单个学生进行预测。

### 5.1 选择任务类型

- **成绩预测（回归）** — 预测学生最终成绩 G3（0–20 分）
- **风险预警（分类）** — 预测学生是否属于高风险（G3 < 10）

### 5.2 选择模型

| 模型 | 说明 |
|------|------|
| 随机森林 | 综合性能最好，推荐首选 |
| SVM | 支持向量机，适合小样本 |
| 决策树 | 可解释性强 |
| 线性回归 / 逻辑回归 | 基线模型，速度最快 |

### 5.3 填写特征参数

**成绩特征：**
- G1：第一次期中成绩（0–20）
- G2：第二次期中成绩（0–20）
- 学习时长：1–4
- 不及格次数：0–4
- 缺勤次数：0–93

**个人信息：**
- 年龄：15–22
- 母亲/父亲教育程度：0–4
- 性别：M / F
- 住址：城市 U / 农村 R
- 家庭规模：LE3 / GT3
- 课外活动：是 / 否

### 5.4 执行预测

点击「开始预测」按钮，右侧面板显示结果：

- **成绩预测**：显示预测的 G3 分数及进度条
- **风险预警**：显示「高风险」或「低风险」标签及风险概率
- **特征重要性图**：展示各特征对预测结果的影响权重（随机森林/决策树有此项）

## 6. 批量预测

点击左侧导航「批量预测」进入，对多个学生同时预测。

### 6.1 配置参数

选择任务类型（成绩预测 / 风险预警）和模型。

### 6.2 上传文件

1. 点击上传区域或将文件拖入
2. 支持 `.csv` 和 `.xlsx` 格式
3. 文件需包含以下列（顺序不限）：

```
age, medu, fedu, studytime, failures, absences, g1, g2, sex, address, famsize, activities
```

可使用项目根目录的 `sample_batch.csv` 作为格式参考。

### 6.3 查看结果

上传后自动预测，结果表格展示：
- 原始特征列
- **预测值**：成绩数值 或 高风险/低风险标签
- **概率**：分类任务显示风险概率

### 6.4 下载结果

点击右上角「下载结果」按钮，将结果导出为 CSV 文件。

## 7. 预测历史

点击左侧导航「预测历史」进入，查看所有历史预测记录。

### 筛选

- 下拉选择「成绩预测」或「风险预警」，点击「查询」过滤
- 留空则显示全部记录

### 字段说明

| 字段 | 说明 |
|------|------|
| ID | 记录编号 |
| 任务 | 成绩预测 / 风险预警 |
| 模型 | 使用的模型名称 |
| 预测结果 | 分数或风险等级 |
| 概率 | 分类任务的置信概率 |
| 学生ID | 关联的学生（批量预测为空） |
| 时间 | 预测执行时间 |

## 8. 模型对比

点击左侧导航「模型对比」进入，比较各模型的性能指标。

### 切换任务类型

顶部 Tab 切换「成绩预测（回归）」和「风险预警（分类）」。

### 指标说明

**回归指标：**

| 指标 | 说明 | 越小越好 |
|------|------|----------|
| MAE | 平均绝对误差 | ✓ |
| RMSE | 均方根误差 | ✓ |
| R² | 决定系数，越接近 1 越好 | — |

**分类指标：**

| 指标 | 说明 |
|------|------|
| Accuracy | 准确率 |
| Precision | 精确率 |
| Recall | 召回率 |
| F1 | F1 分数（精确率与召回率的调和均值） |
| ROC-AUC | ROC 曲线下面积，越接近 1 越好 |

### 特征重要性图

展示随机森林模型中各特征的重要性排名，数值越大说明该特征对预测结果影响越大。

## 常见问题

**Q: 点击预测后提示「请先训练模型」**

需要先执行模型训练：
```bash
docker compose exec backend python ml/train_all.py
```

**Q: 数据概览全部显示 —**

数据库中没有学生数据，先在「学生管理」中导入数据。

**Q: 批量导入失败**

检查 CSV 文件编码是否为 UTF-8，列名是否与要求一致。

**Q: 重置数据库**

```bash
docker compose down -v
docker compose up -d
```
